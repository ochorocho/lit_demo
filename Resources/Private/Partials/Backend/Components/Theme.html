typo3-backend-color-scheme-switch

Three-way switch for the backend color scheme: light, dark, or auto (follows the OS). Persists the choice on the current backend user via the user-settings endpoint.

The component is designed to live inside the user-toolbar dropdown (#typo3-cms-backend-backend-toolbaritems-usertoolbaritem), so two pieces of styling are inherited from its production ancestor and would otherwise be missing here:

  • The toggle / chevron buttons rely on color-scheme: only dark propagated from .scaffold-header — without it, their :hover background sits on a near-white panel and is barely perceptible.
  • The <button class="dropdown-item"> entries (Light / Dark / Auto) use --typo3-dropdown-item-hover-bg, which is only declared inside .dropdown-menu; without that ancestor the hover background falls through to transparent.

To make the demo accurately reflect production styling we re-declare the relevant variables on the switch element itself via the .lit-demo-content typo3-backend-color-scheme-switch selector in lit-demo.css.
